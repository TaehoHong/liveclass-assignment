## 설계 결정과 이유

### 정산 조회와 정산 실행 분리
- 정산 조회는 "현재 보여줄 금액"을 제공하고, 정산 실행은 "상태가 있는 정산 데이터"를 생성하는 별도 흐름으로 본다.
- 운영자가 정산을 실행하면 `PENDING` 상태로 `settlement`가 생성되고, 크리에이터 확인 후 `CONFIRMED`, 운영자 지급 후 `PAID`로 진행하는 정책을 후속 기능의 기준으로 둔다.
- 마감된 월의 정산 조회는 저장된 정산 데이터만 반환한다.
- 현재월은 미마감 월이므로 저장된 정산 데이터가 없으면 예상 정산액을 계산해 반환한다.
- 월 변경 직후 전월 정산이 아직 생성 중이면 전월 정산 조회는 빈 배열로 응답한다.
